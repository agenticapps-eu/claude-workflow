# P6.5 — Tracking issues (DEFERRED)

Per user direction at v1.14.0 ship time: the sibling repos
(`pi-agentic-apps-workflow`, `codex-workflow`) have a lot of older
spec-version catch-up still pending. A single 0.4.0-absorption
issue would land alongside larger backlog work that doesn't yet
have a tracking shape. Filing held; drafts preserved here for the
operator to file when the sibling repos are ready for per-version
tracking issues.

The 1.14.0 release does NOT depend on these issues existing — the
local ADR 0024 + cross-repo workflow-core PR #10 are the
load-bearing artefacts. The tracking issues are a discoverability
aid, not a gate.

---

## Draft 1 — `agenticapps-eu/pi-agentic-apps-workflow`

**Title:** Absorb agenticapps-workflow-core 0.4.0 (claude-workflow v1.14.0 parity)

**Body:**

```markdown
Tracking issue for absorbing `agenticapps-workflow-core` spec **0.4.0** into the Raspberry Pi flavor of the workflow scaffolder. The Claude flavor (`claude-workflow`) shipped 0.4.0 absorption in v1.14.0; this issue tracks the analogous pi-side adoption.

## Spec 0.4.0 sections to absorb

- [ ] **§11 — Coding Discipline (NON-NEGOTIABLE)** — verbatim canonical block (four rules: Think Before Coding / Simplicity First / Surgical Changes / Goal-Driven Execution) MUST be injected into the host's primary project-instruction file behind a provenance-managed anchor (`<!-- spec-source: agenticapps-workflow-core@0.4.0 §11 -->`). See claude-workflow `migrations/0014-inject-spec-11-coding-discipline.md` for the reference implementation pattern (vendored block at `templates/spec-mirrors/11-coding-discipline-0.4.0.md`, byte-identical to workflow-core's `spec/11-coding-discipline.md` canonical fence).
- [ ] **§12 — Branchy workflows (Mermaid)** — SHOULD-level convention for branchy paragraphs (≥2 decision branches AND ≥1 cycle/fallback). MUST satisfy in every host SKILL.md/AGENTS.md/contract spec file newly authored at or after 0.4.0 adoption. MAY convert pre-existing files opportunistically (bulk conversion not required). See claude-workflow `.planning/phases/20-spec-0.4.0-absorption/P3-AUDIT-LOG.md` for the audit-trail pattern.
- [ ] **§13 — Declare-first TypeScript** — host MUST ship the `ts-declare-first` skill scaffold (three atomic commits: Phase 1 declaration, Phase 2 failing tests, Phase 3 implementation; skill REFUSES to bundle phases). Implicit-trigger wiring for TS-primary projects is future work — opt-in/explicit-invocation is the v0.4.0 floor. See claude-workflow `ts-declare-first/` and `migrations/0015-add-ts-declare-first-skill.md` for the reference implementation.
- [ ] **ADR-0015 — Secret-scanner choice (RATIFIED)** — Status: Proposed → Accepted. Decision: STAY on gitleaks. Locked-rule benchmark fired STAY via ties on TP recall + FP count criteria; secondary credibility signal from gitleaks 8.30.1 decoding inline base64 by default (betterleaks 1.3.0 does not). 12-month re-evaluation reminder in Consequences. See cross-repo PR https://github.com/agenticapps-eu/agenticapps-workflow-core/pull/10 and claude-workflow's local mirror `docs/decisions/0024-secret-scanner-choice.md`. **No CI fragment** ships in claude-workflow (STAY = downstream projects choose locally); pi-side decision can mirror or revise.

## Cross-references

- Spec source: https://github.com/agenticapps-eu/agenticapps-workflow-core (sections 11, 12, 13 + adrs/0015)
- Reference adoption PR (claude flavor): claude-workflow's v1.14.0 PR (to-be-opened post merge of cross-repo PR #10)
- Reference CHANGELOG entry: `claude-workflow/CHANGELOG.md` `[1.14.0] — 2026-05-21`

## Acceptance criteria

- [ ] pi-side `implements_spec` bumped to `0.4.0` on the skill manifest.
- [ ] §11 canonical block present in pi-side default CLAUDE.md / equivalent.
- [ ] §12 audit log for any new pi-side SKILL.md files; pre-existing deferred per waiver.
- [ ] `ts-declare-first` skill scaffold present in pi-side bundle (vendored or symlinked).
- [ ] ADR-0015 decision (STAY) acknowledged in pi-side ADR mirror or referenced by upstream.
```
