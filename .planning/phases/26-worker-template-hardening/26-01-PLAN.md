    > "Each Worker invocation executes in its own V8 isolate. Module-level mutable state is reset between invocations because each invocation loads a fresh module instance. The `init()` function is therefore safe to call once per invocation without global mutation risk across requests."

    **Decision section MUST state explicitly:**
    1. The cf-worker / cf-pages / openrouter-monitor `init()` is intentionally NOT guarded by an `initialized` flag — last-call-wins semantics within the isolate is the contract.
    2. The supabase-edge `init()` IS guarded by `let initialized` + `if (initialized) return` — first-call-wins semantics is the contract on Deno-edge.
    3. **D-02b — supabase-edge extra out-of-band state**: The invariant explicitly covers supabase-edge's additional module-level state:
       - `let initialized: boolean = false` — an idempotency-enforcing flag (NOT a request-scoped value); the invariant says it is reset between isolate instances, so the first-call-wins guard is sound per-invocation.
       - `let _testEnv: InitEnv | null = null` — a test-only seam set via the existing `_resetForTest(env)` export; the invariant covers it as out-of-band state that is NEVER set in production code paths. Production never observes a non-null `_testEnv` because the production code path never calls `_resetForTest`.
       Both pieces of extra state are covered by the same isolate-per-invocation guarantee that covers `serviceName`, `deployEnv`, and `registry`. The ADR names them by name so future readers understand the full scope of "module-level mutable state" the invariant protects.
    4. A test (D-02a) per stack asserts the contract: `init() called twice within isolate yields deterministic singleton state`.

    **Consequences section MUST cover:**
    - What this ADR locks: the contract that future refactors (AsyncLocalStorage, per-request closure — both deferred per 26-CONTEXT §Deferred Ideas) must honour or replace explicitly.
    - When the invariant breaks: if Cloudflare ever ships concurrent invocations sharing one isolate, OR if the wrappers are ported to a long-lived Node/Deno process with shared module state, the invariant breaks and a refactor is required. The supabase-edge `_testEnv` seam additionally requires that production code never call `_resetForTest` — the JSDoc `@internal` marker is the documentation that enforces this.
    - Risk today: LOW — Cloudflare's published runtime model guarantees isolate-per-invocation; Deno-edge functions are short-lived per request.

    **Rejected alternatives section MUST cover:**
    - AsyncLocalStorage refactor (over-engineering for latent issue; deferred per CONTEXT §Deferred Ideas)
    - Per-request closure with explicit InitContext parameter (largest API surface change; deferred)
    - Defer entirely (rejected — the invariant is undocumented today, which itself is the bug DEF-3 names)
